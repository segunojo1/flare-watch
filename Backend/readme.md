# FlareWatch API: Frontend Integration Guide

This document outlines the telemetry data pipeline for the FlareWatch 3D Globe. The backend is fully automated, pulling near real-time (NRT) satellite thermal data from the NASA FIRMS network (Suomi NPP / VIIRS), filtering it through regional African oil-block geospatial polygons, and serving it instantly.

## The Endpoint

**URL:** `GET https://flarewatcher.onrender.com/api/v1/telemetry/live`  
**Method:** `GET`  
**Rate Limit:** None (Data is cached on the backend and refreshes automatically every 12 hours).

---

## Data Structure

The endpoint returns a JSON object with two main keys: `meta` (context) and `telemetry` (the actual array of flares). 

**Performance Note:** The backend processes ~ 5,000 raw thermal anomalies daily but uses strict geospatial filtering to drop agricultural bushfires. You will receive a clean payload of roughly **~400 to 500 verified gas flares**. This is well within the DOM budget for `react-globe.gl` `htmlElementsData` and will maintain a smooth 60 FPS.

## Property Breakdown & Data Dictionary

Here is exactly what each piece of data means and how it should be used in the UI.

### The `meta` Object (Contextual Data)
Use this data to populate the corner of your UI dashboard to prove the data is live and legitimate.
* **`satellite` & `dataset`:** The specific NASA satellite and instrument used to capture the data.
* **`timestamp`:** The exact UTC time the backend last fetched and filtered this data.
* **`region`:** The geographic scope of the telemetry points.

### The `telemetry` Array (The Gas Flares)
This is the array you will loop through to render points on the globe.
* **`id`:** A unique identifier generated by the backend. **Always use this as your React `key`** when mapping over the array to prevent rendering glitches.
* **`lat` & `lng`:** The exact latitude and longitude coordinates. Plug these into `react-globe.gl` to plot the point on the 3D sphere.
* **`radiant_heat_mscf`:** The estimated intensity/volume of the flare. 
    * *UI Tip:* Use this value to dynamically scale your CSS `width`/`height` or `transform: scale()`! A flare with `150 mscf` should have a massive red glow, while `15 mscf` should be a small orange dot.
* **`attribution`:** The corporate ownership data for that specific coordinate.
    * **`block`:** The geographical oil basin or block (e.g., Niger Delta).
    * **`operator`:** The company operating that block (e.g., NNPC / Shell). 
    * *UI Tip:* Use `operator` and `block` to populate tooltips or side-panel cards when a user clicks on a specific flare dot!

## React-Globe.gl Integration Example

Here is a quick snippet of how to wire the JSON into the Globe using HTML markers:

```jsx
import React, { useEffect, useState } from 'react';
import Globe from 'react-globe.gl';

const FlareGlobe = () => {
  const [flares, setFlares] = useState([]);

  useEffect(() => {
    // Fetch the live, filtered NASA data
    fetch('[https://flarewatcher.onrender.com/api/v1/telemetry/live](https://flarewatcher.onrender.com/api/v1/telemetry/live)')
      .then(res => res.json())
      .then(data => setFlares(data.telemetry))
      .catch(err => console.error("API Error:", err));
  }, []);

  return (
    <Globe
      globeImageUrl="//[unpkg.com/three-globe/example/img/earth-dark.jpg](https://unpkg.com/three-globe/example/img/earth-dark.jpg)"
      htmlElementsData={flares}
      htmlLat="lat"
      htmlLng="lng"
      htmlElement={d => {
        const el = document.createElement('div');
        
        // Dynamic sizing based on heat (min 10px, scales up)
        const size = Math.max(10, d.radiant_heat_mscf / 5); 
        
        el.style.width = `${size}px`;
        el.style.height = `${size}px`;
        el.style.background = 'radial-gradient(circle, rgba(255,50,0,0.8) 0%, rgba(255,50,0,0) 70%)';
        el.style.borderRadius = '50%';
        el.style.cursor = 'pointer';
        el.style.pointerEvents = 'auto';
        
        // Add click event to trigger UI updates
        el.onclick = () => {
          console.log(`Clicked on ${d.attribution.operator} flare in ${d.attribution.block}!`);
          // Trigger your state update here to show the AI Sentinel card
        };
        
        return el;
      }}
    />
  );
};

export default FlareGlobe;
```
